About

I was working on a new site for a client who needed a simple way to post articles and already had access to a server. Instead of using Wordpress, which would have been more complicated and bloated process than the user needed, I decided to build my own. Using Python and MySQL I decided to make my own article app to have at my call whenever I needed it.

Starting Point

Choosing what frameworks and systems to use took a little thinking.

  • Python Flask - flexible, minimalist, pretty easy URL routing.
  • MySQL - I'd worked with other SQL databases before, but hadn't done any deployment with MySQL, why not learn.

MySQL

Setting up MySQL in OSX isn't too tricky and once it's up running things go pretty smoothly. I only needed two tables - users and articles - to handle all the data we'd need.

Then I had to plan out how the site map would work:

  • Users - I didn't want give posting-status to just anybody. So I decided on an account system that only allowed users to post articles and also create new accounts.
  • Articles - To keep it simple I decided on a system that posted the newest article to the front page and listed all older articles, available to everyone.

Keeping users exclusive

By only allowing current users to create new user accounts, it acts as a way to buffer random users spamming articles to the app. Any current user can create an account for a friend, pass along the information, and the friend can then log in and change their password. After adding routes and a dashboard for users to edit, add, and delete their articles everything was set.

Migration!

Launching to Heroku, which doesn't have MySQL natively, meant a few changes had to be made. Addition of CloudDB and migration and all data/tables to the new server things started to look ok for deployment. If you're pushing a flask app to heroku yourself, make sure you create a Profile.txt with web gunicorn app:app in your root folder and don't forget your requirements.txt!
