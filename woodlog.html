Wood Log was supposed to be a simple solution to a simple problem. In the beginning I wanted a way to log the hikes Iâ€™d been on, a running checklist, basically. After adding a few features (and a little expansion) I wound up with a minimal, account-based system.

User Page

The user page needed to have a way for users to log their finished hikes, add future hikes, and sort between the two. After some input from people using the app, more activity options, a rating system, and a distance/difficulty rating were added.

Accounts

Every account-based system has to think about security. Keeping in line with everything being minimal and simple as possible, I went with MongoDB and B-crypt for hashing. Along with the standard form-clearing measures.
